# Правила анализа

Когда пользователь просит **проанализировать** что-либо (рынок, код, архитектуру, модуль, данные):

## Обязательный процесс

1. Провести детальный анализ запрошенной темы
2. Сохранить результат в `.claude/analysis/YYYY-MM-DD-<slug>.md`
3. Формат имени файла: дата + краткое описание через дефис (например `2026-03-03-btc-market-analysis.md`)

## Шаблон файла анализа

```markdown
---
type: analysis
topic: <тема анализа>
date: YYYY-MM-DD
status: completed
tags: [crypto, forex, code, architecture, performance, security]
---

# Анализ: <тема>

## Контекст
<что анализировалось и зачем>

## Данные
<факты, метрики, наблюдения — с источниками>

## Выводы
<ключевые находки, пронумерованный список>

## Рекомендации
<конкретные действия, если применимо>
```

## Типы анализа

- **Рыночный** — индикаторы, тренды, уровни, новости (делегируй analyst)
- **Кодовый** — качество, паттерны, проблемы, покрытие тестами
- **Архитектурный** — структура, зависимости, точки расширения
- **Безопасности** — credentials, уязвимости, audit
- **Производительности** — bottlenecks, оптимизации

## Правила

- Каждый анализ — отдельный файл в `.claude/analysis/`
- Всегда указывать источник данных
- Выводы подкреплены фактами, не домыслами
- После анализа — кратко сообщить пользователю ключевые выводы и путь к файлу
